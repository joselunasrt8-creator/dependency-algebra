# SYNAPSE Highest-Leverage Plan Audit

## 1. Final Determination

READY_WITH_BLOCKERS

The repository has the core technical capability required for the smallest external workflow experiment: accepted topology JSON can be compiled through the documented CLI into a deterministic structural artifact containing classification, failure surface, redundancy map, annihilation conditions, provenance, and artifact hash. The launch blocker is not analysis capability. It is experiment packaging: there is no single external-facing decision example that contains topology input, workload, candidate removal, generated artifact, human-readable interpretation, and explicit before/after engineering decision.

- Current implemented predicate directly matches the candidate wedge: removing a workload candidate set and checking whether directed root-to-target reachability remains.
- Topology input shape is documented and fixture-backed.

## 4. Launch-Blocking Gaps

### Gap 1: No complete external-facing decision example

- Evidence: fixtures exist for minimal valid, redundant, degraded, and null cases, and the fixture catalog names `fixtures/null/minimal-null.json` as a candidate-removal case that collapses workload reachability. However, there is no single documented example that bundles the input, generated artifact, human-readable interpretation, and clear before/after engineering decision.
- Affected files: add one documentation file under `docs/` and, optionally, add a generated example artifact under a deterministic fixture/example path if the project wants the artifact committed rather than generated by command.
- Smallest correction: create one bounded example named around the real question, using `fixtures/null/minimal-null.json`, and include the exact CLI command plus an interpretation table: workload `serve-api`, candidate `api`, decision before “assume removing `api` is acceptable or needs review,” evidence “candidate set is a structural dependency because reachability after projection is empty,” decision after “investigate or revise removal because workload reachability collapses.”
- Validation command: `python -m dependency_algebra.cli compile --input fixtures/null/minimal-null.json --output /tmp/synapse-null-artifact.json`.
- Why necessary: the experiment measures whether the artifact changes or sharpens an independent engineer's decision. Without a complete example, the first participant must infer the decision frame from fixture names and raw artifact fields, which adds avoidable overhead and confounds usability with analysis value.

### Gap 2: Five-minute path lacks an explicit install-from-public-location command

- Evidence: README documents command shape and installed console script, while package metadata declares the package name and script. The repository CI builds and installs the wheel locally. The README does not show an exact public install command such as `pip install synapse-structural-analysis` or `pip install git+<public-url>`.
- Affected files: `README.md`.
- Smallest correction: add a short “Five-minute external path” section with the current supported install command from the repository's actual public distribution channel, followed by copying one fixture or using a local topology file and running the compile command.
- Validation command: run the exact documented install sequence in a clean virtual environment, then run `synapse compile --input fixtures/basic.json --output /tmp/synapse-artifact.json` or the equivalent command against a copied topology file.
- Why necessary: an independent engineer cannot reliably install from “the documented public path” unless that path is explicit and copy/pasteable.

## 6. External Experiment Protocol

- Participant: one engineer who did not author SYNAPSE and owns or understands one real topology-changing PR.
- Topology: one JSON topology containing components, directed edges, one workload, one target, and the component or set being removed.
- Initial assumption: written before running SYNAPSE, for example: “Removing component X should not collapse workload Y reachability because path Z remains.”
- Command: `python -m dependency_algebra.cli compile --input topology.json --output synapse-artifact.json`.
- Artifact: `synapse-artifact.json`, retained in CI or attached to the PR for human review.
- Interpretation: participant reads classification, dependency lattice, failure surface, redundancy map, annihilation conditions, provenance, and artifact hash.
- Decision before: approve, investigate, or revise based on pre-existing review knowledge.
- Decision after: approve, investigate, revise, or keep the same decision with sharper rationale after reading the artifact.
- Final determination labels:
  - SUPPORTED: decision influence observed.
  - NOT_SUPPORTED: artifact was understood and trusted but did not affect reasoning or action.
  - BLOCKED: input, installation, or usability failure prevented testing decision influence.
  - INDETERMINATE: observation was insufficient.

This creates the most new information because it tests the full wedge without adding architecture or policy: topology-changing PR, SYNAPSE analysis, deterministic artifact, engineer review, and a human decision.
